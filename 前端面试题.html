重绘 重排

浏览器运行机制

  1. 构建dom 树 解析html 文档 将标签转换成 dom树种的DOM node, 生成DOM tree

  2. 解析渲染树 解析对应的css 样式文件(包括js生成的) 构建render tree , render tree 中不包含隐藏的节点 head节点

  3. 布局渲染树 从根节点递归调用 计算每个节点大小 位置 给出每个节点在屏幕上的精确坐标

  4. 绘制渲染树 遍历渲染树 使用ui层绘制每个节点

重绘

当盒子位置 大小及其他属性(字体 颜色…) 确定以后 ,浏览器会吧这些属性按各自特性绘制一遍 将内容呈现在页面上
重绘 是一个元素外观改变所触发的浏览器行为 浏览器会根据元素的新属性重新绘制, 使元素呈现性的外观 ()

重绘的触发条件 (background-color color…)

回流(重排/重构)

当渲染树种的元素规模尺寸 布局 等改变页面需要重新构建 这就称为回流 每个页面第一次加载必定回流

触发回流条件 (任何页面布局和几何属性的改变都会触发回流(重排)
1. 页面渲染初始化
2. 添加或者删除可见元素
3. 元素位置改变,或者使用动画
4. 元素尺寸改变(resize事件改变)
5. 填充的内容改变 比如文本改变 或者图片改变 引起计算值宽高改变
6. 读取某些元素属性(offsetLeft/Height/Width …)

重绘 回流关系

回流必然触发重绘 但重绘不一定触发回流

优化

  1. 直接改变classname
  2. 使用决定定位 或固定定位 脱离文档流 ,使它的变化不会影响到其他元素
  3. display:none 文档渲染完毕后再设置 display:block 会触发两次重排 重绘
  4. 如果需要创建多个DOM节点 可在创建万完毕后一次性的加入document
  5. cloneNode 和 replaceChild 引发一次重绘重排

一次完整的http 服务过程

当我们在web浏览器中输入 www.baidu.cm 具体发生了什么

  1. 对 www.baidu.com 这个网址进行域名DNS解析,得到对应的ip地址

  2. 根据ip 找到对应服务器 ,发起tcp 三次握手

  3. 跟服务器建立TCP连接后发起http请求

  4. 服务器响应htpp请求 浏览器得到html代码

  5. 浏览器解析html代码 并请求html中的资源

  6. 浏览器渲染页面呈现给用户

  7. 服务器关闭TCP连接

DNS怎么找到域名的

​ DNS域名解析采用递归查询方式 先去找 DNS缓存-> 根域名服务器->下一级

为什么http协议要基于TCP实现

TCP是一个端到端的可靠的面向连接的协议 http基于传输层TCP 协议不用担心数据传输的各种问题(当发生错误时,会重传)

防抖 节流

防抖

​ 只有任务触发间隔超过n秒才会执行 比如 input 输入搜索

const debounce = (fn,delay:number){
    let inDebounce
    return function(...args){
        const context = this
        clearTimeout(inDebounce)
        inDebounce = setTimeout(()=>{
            fn.apply(context,args)
        },delay)
    }
}

节流

频繁触发 n秒内只能执行一次 通常在scroll 滚动事件中使用

 const throttle = function (fn, delay) {
            var timeout = null
            return function (...args) {
                var context = this
                if (!timeout) {
                    timeout = setTimeout(_ => {
                        timeout = null
                        fn.call(context, args)
                    }, delay)
                }
            }
        }

        // 使用
        function fnc(e) {
            console.log(e);
        }
        // 处理函数
        const resultFn =  throttle(fnc, 1000)
        setInterval(_ => {
           resultFn(new Date())
        }, 10)
